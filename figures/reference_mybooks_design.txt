1 引言
  1.1 编写目的

  需求规格说明书中，已经将用户对本系统的需求做了详细叙述及的阐明，本
阶段是在系统的需求分析的基础上，对系统做概要设计。主要解决实现该系统需
求的程序模块设计问题，包括如何把该系统划分成若干个模块、决定各个模块之
间的接口、模块之间传递的信息，以及数据结构、模块结构的设计等。本文档也
是一种规范，旨在推动软件工程的规范化，使设计人员遵循统一的概要设计书写
规范，节省制作文档的时间，降低系统实现的风险，做到系统设计资料的规范性
与全面性，以利于系统的实现、测试、维护、版本升级。本文档供设计人员、开
发人员参考。


  1.2 背景

  图书馆是学习的一个重要资源，对于智慧校园的建设有着重要作用。我国很
多高校的图书馆都有丰富的藏书，对大量藏书的管理成为了急需解决的问题。为
了全面提升优质服务效率，充分利用图书馆丰富的资源，特引入图书管理系统，
打造智慧图书解决方案，推动图书馆的智能化建设。图书馆管理系统可以在很大
程度上方便读者查看和利用图书馆的图书资源，通过智能化管理，读者可以自助
使用图书资源，方便读者使用图书馆资源。智能图书管理系统同时方便了图书管
理员的管理工作，相较于传统的管理方式，图书管理系统提供了智能的管理方式，
图书管理员可以利用图书管理平台及时查看图书的相关情况，对大量藏书进行系
统化的管理，从而使图书资源得到充分利用。

2 总体设计
  2.1 需求规定

  基于需求分析，此处对校园智慧图书管理系统的实现做相应的规定，其中包
含了对功能、性能、兼容性、界面要求等需求的规定。

2.1.1 对功能的规定
  书刊管理。能够对各种类型书刊的基本信息进行分类管理。能够对书刊借阅
信息进行管理，包括在馆数量、借还信息、读者信息、归还时间、是否超期与罚
金等信息。
  会员管理。会员个人信息、用户密码、可借书数量、免费租赁天数、押金、
租金和滞纳金等信息的管理。
  财务管理。支持押金收款、退款管理，预存款收款、退款操作，支持图书超
期罚款、丢失罚款、损毁罚款操作。
  查询统计。支持对书刊借阅情况、书刊数据盘点、借阅排名、逾期记录、充
值记录、消费记录等信息的查询与数据报表的生成与导出。
  自助借还书。支持读者自助借书、自主还书、自主续借、书籍预约登记、未
在馆书籍登记申请等操作；支持通过读者卡号或书籍条码等书籍信息进行快速借
还书操作。
  无馆藏图书申请。馆内若未收录读者希望借阅的书籍，支持读者填写书籍名
称、作者、出版社（选填）、语种（选填）等信息进行申请。
  信息查询。支持会员对账户个人信息的查询与修改；支持会员对账户借还书
详情、可借书数量、免费租赁天数、押金、租金和滞纳金等信息的查询；支持会
员对书刊馆藏位置、在馆状态、书刊分类等基础信息进行查询。

2.1.2 对性能的规定
  精度。保证查询的查全率和查准率为 100%，所在相应域中包含查询关键字
的记录都能查到，所有在相应域中不包含查询关键字的记录都不能查到。
  时间。系统对绝大多数功能的响应时间在 1-2 秒内。
  灵活性。采用前后端分离的设计，前端设计可以在 GUI.py 文件中进行相应
的修改，如添加相应按钮功能等。系统采用模块化程序设计方法，既便于系统功

2.1.4 界面要求
   1. 登录界面（注册、登录）
   注册：普通用户填写 id、昵称与密码来创建用户；登录：用户输入正确的 id
和密码后即可成功登录并跳转操作界面，否则提示错误信息。
   2. 主界面（管理员界面、用户界面）
   管理员界面。由管理员账号登陆后，首页显示书刊管理模块的功能，从菜单
栏选取其他模块并跳转，页面中的搜索框通过输入书籍关键词或书刊码搜索目标
书籍，通过输入昵称或用户 id 搜索目标用户。实现以下要求：
   书刊管理：包括导入书刊，点击后跳转书刊信息填写界面，填写完成并提交
即算书刊导入成功。
   会员管理：通过输入昵称或用户 id 搜索目标用户，进入用户详情页后可浏
览用户详细信息，或对相关信息或密码进行修改删除等操作。
   财务管理：显示当前押金收取状况，退款进度，用户预存款的收款退款情况
与罚款收取情况，其中罚款模块包括图书超期罚款、丢失罚款与损毁罚款。
   查询统计：主界面显示书籍借阅情况、书刊数据盘点与借阅排名，在功能栏
点击跳转查看其他信息，如逾期记录、借阅排名、充值记录、消费记录等信息。
所有数据支持生成报表并导出。
   用户界面。由用户账号登陆后，首页显示个人借还书情况、可借书数量、免
费租赁天数、押金、租金和滞纳金等信息。搜索栏可通过关键词或书刊码搜索目
标书籍信息。功能栏包括自助借书、自主还书、自主续借、书籍预约登记、未在
馆书籍登记申请等按钮。点击右上角个人头像可跳转至个人信息页，可以对个人
信息进行浏览或修改。

   2.4 设计原则与思想

2.4.1 设计原则
  1. 先进性：系统在整体方案及各个子模块的设计时采用当前先进成熟的技
     术和设备。
  2. 安全性：系统的设计与规划要从网络设计、软件设计、数据库设计以及
     设备选型等方面通盘考虑。要确保系统长时间可靠、稳定的运行。要避
     免非法人员假冒，给图书馆带来一定的经济损失。
  3. 实用性：UI 界面美观、逻辑合理、交互友好，在设备的采购上要选取合
     适的产品型号，确保系统长时间的稳定运行，系统运行顺畅无卡顿，无
     闪退等严重 BUG。系统稳定性好，崩溃率低。系统响应速度快。
  4. 方便性：要有完善的系统运行保障机制，在系统的操作上要求简单明了，
     借阅者与图书管理人员都很方便的使用该系统。
  5. 可扩展性及易维护性：系统在功能模块上面要预留相应的接口以方便系
     统不仅在今后的更新、管理、维护方面代来极大的便利还要保护学校现
     有的投资，弹性可伸缩，系统要具备较高的性价比。
  6. 开放性：考虑到学校的长期利益，不同的产品供应商的机具必须兼容其
     它厂商的设备从而最大限度的保护学校的利益。系统在开发的过程当中
     必须结合国家颁布的相关的标准或者是工业标准。

2.4.2 设计思想
  1. ETC 思想（Easier to change）： 软件应该遵守“高内聚，低耦合”的原
      则，能不断应对需求变化。
  2. DRY 思想（Don't Repeat Yourself）： 软件的最小模块在系统中应当是
      单一的、明确的，不应该重复出现。
  3. 遵循下列特性：可维护性（Maintainability）、可复用性（Reusability）、
      可扩展性（Extensibility / Scalability）、灵活性（flexibility / mobility /
      adaptability）。

4 模块设计
  4.1 用户模块

4.1.1 借阅管理模块
  主要为读者提供自助借阅、自助还书、自助续借、自助预约的服务，具体功
能有：
  ◼ 列出图书信息
  ◼ 申请借书，管理借阅列表
  ◼ 选择还书，并生成扣费类型和扣费金额
  ◼ 续借图书，且不超过两次
  ◼ 预约图书，预约信息汇总在预约列表

4.1.2 用户信息模块
  主要为读者提供个人信息的维护，具体功能为：
  ◼ 保存用户的个人信息，并用于用户行为统计分析
  ◼ 针对用户的信息形成用户画像，并进行个性化推荐

4.1.3 采购申请模块
  主要为读者提供未上架书籍采购申请的服务，具体功能有：
  ◼ 读者填写并提交采购图书信息
  ◼ 整合书籍申购列表并返回给管理员

4.1.4 财务管理模块
  主要对读者的余额、罚款、押金进行管理，具体功能有：
  ◼ 余额的充值与提现
  ◼ 罚款的记录与缴纳
  ◼ 押金的支付与提现

9 安全性设计
   9.1 数据传输安全

  数据传输过程中，安全性十分重要，尤其是对于一些敏感的用户信息，其安
全性保证更应当被重视。
  现代软件开发中，无论是基于敏捷式的软件开发，或者基于前后端分离的软
件开发，都离不开提供对外接口，而请求这些接口数据的安全性应当得到保证。
  为确保数据安全传输，本系统采用了多种方法。
  1. 使用 HTTPS 传输协议
  本系统网络数据传输均基于具备 TLS/SSL 加密层的 HTTPS 协议进行传输。
所有信息都是加密传播，第三方无法窃听。具有校验机制，一旦被篡改，通信双
方会立刻发现。配备身份证书，防止身份被冒充。
  2. 基于身份验证的安全控制。
  为了防止各种假冒攻击,在执行真正的数据访问操作之前,要在客户和数据
库服务器之间进行双向身份验证,比如数据库系统服务器与服务器之间进行数据
传输时,都需要验证身份。
  本系统使用基于 Token 的身份验证方法，在服务端不需要存储用户的登录
记录，大致流程如下：用户登陆时，若身份验证通过，服务端会签发给客户端一
个 Token；客户端收到 Token 后可以将其储存起来，基于该 Token 进行各项操作。
  3. 对于数据的访问控制策略
  在通常的数据库管理系统中,为了防止越权攻击,任何用户不能直接操作库
存数据。用户的数据访问请求先要送到访问控制模块审查,然后系统的访问控制
模块代理有访问权限的用户去完成相应的数据操作。
  用户的访问控制有两种形式:自主访问授权控制和强制访问授权控制。其中
自主访问授权控制由管理员设置访问控制表,此表规定用户能够进行的操作和不
能进行的操作。而强制访问授权控制先给系统内的用户和数据对象分别授予安全
级别,根据用户、数据对象之间的安全级别关系限定用户的操作权限。

   9.2 系统应用安全

9.2.1 认证机制
  登录时需要输入系统分配的用户名和密码，若连续输入错误次数达到系统阈

9.2.2 日志管理
  系统详细记录用户登录、登出、处理视频、分析结果等重要操作日志，便于

   9.3 数据存储安全

  设计数据存储安全的目的是确保数据的完整性，防止数据被破坏或丢失，保
证数据的保密性、完整性、可用性、真实性。采取的数据存储安全的策略是：
  1. 监测活动
  全年全天候对用户的行为进行检测。只有经过授权的设备才可使用识别服务
并访问基础信息库。对于单个管理员，检测事件日志并定期进行审计。
  2. 访问控制
  对数据的访问、维护权限只能授予那些需要访问数据的人。
  3. 维护信息
  保护所有开发过程信息，不使用不受控制的移动存储设备，如闪存驱动和
DVD 等，让大量数据处于威胁之中。
  4. 数据处理政策
  实施严格的安全政策，包括数据是如何处理的、如何访问和转移等。让每一
个开发成员都被强有力的可执行的安全政策限制，加强开发人员对安全问题的认
知，这将能够有效提高存储安全水平。
  5. 备份数据
  使用差异备份方式，每次备份时对上次完全备份之后所有发生改变的数据进
行备份，并定期检查备份，并且防备备份遭到破坏。
  6. 病毒防范

12 缺陷分析和系统优化方案
 12.1 算法层面

1. 统计算法
 （1）缺陷分析
  当前的查询统计仅支持对书刊借阅次数、书刊数据盘点、充值记录、消费
  记录等信息的朴素查询与数据报表的生成与导出。
 （2）优化方案
  采用金蝶云苍穹平台的卡片视图和轻分析视图，对统计结果进行可视化展
  示。同时，设置热门书单排行榜、读书者热榜等榜单，对统计结果加以利
  用，进一步服务用户，并在年度报表生成中为用户生成相关数据并提供。
2. 推荐算法
 （1）缺陷分析
  目前仅支持根据读者借阅次数分析借阅偏好，预测喜好的书籍并随机推荐
  该类书籍。
 （2）优化方案
  采用 FM(Factorization Machine)作为推荐算法，通过引入二阶特征，并利
  用正定矩阵和稀疏矩阵的性质将二阶特征的计算降低至线性复杂度。通过
  多维分析爱好的书籍分类，并提取该类书籍中借阅次数 Top10 的书单进行
  推荐。


 12.2 系统及 UI 层面
